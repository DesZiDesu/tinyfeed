TinyStream (ไลฟ์สตรีม)
โหมดอัตโนมัติจะเรียก AI เรื่อยๆ ตอนเปิดหน้าไลฟ์ (เปลือง token) · ตั้งวินาทีสูงขึ้นเพื่อประหยัด
TinyConnect (แชต)
เมื่อคุยในแชทหลักครบตามจำนวน ตัวละครหลักจะส่งข้อความหาคุณเองในแอปแชต พร้อมแจ้งเตือน (นับจากตอนเปิดแชท)
โมเดล / API
ข้อมูลการ์ดตัวละคร + persona ส่งเต็มเสมอ · World Info ถ้ายาวมากค่อยตั้งลิมิตกัน prompt บวม แยกโมเดลของ TinyFeed ออกจากโมเดลตอบ RP ได้ · ต้องสร้าง connection profile ใน Connection Manager ของ ST ก่อน · โหมด "API หลัก" ได้ context เนื้อเรื่องอัตโนมัติ ส่วนโมเดลแยกเราจะแนบบทสนทนาล่าสุดให้ตามจำนวนนี้ (มากขึ้น = เข้าใจเนื้อเรื่องดีขึ้น แต่เปลือง token)
คำสั่งระบบ / Jailbreak
แนบเป็นคำสั่งระบบบนสุดทุกครั้งที่ TinyFeed เรียก AI (ฟีด · แชต · ข่าว · ไลฟ์) · จำเป็นมากถ้าใช้ "โมเดล/API แยก" เพราะโมเดลแยกไม่ได้รับเจลเบรกจาก preset ของ ST เลย · ปล่อยว่าง = ไม่แนบ

โพสต์จากตัวละคร (AI)
โพสต์ที่ AI สร้างจะสุ่มไลค์ในช่วงนี้ · เลขเยอะจะย่อเป็น K/M ในฟีด แต่โชว์เต็มเมื่อกดเข้าโพสต์ แนบโพสต์ล่าสุดเป็นบริบท → AI ไม่โพสต์ซ้ำ + อ้างอิงได้ · 0 = ไม่แนบ (ยิ่งเยอะยิ่งเปลือง token)
NPC ประจำ (แชทนี้)
กำหนดตัวละครประจำโลกของแชทนี้ · AI จะเลือกโพสต์จากรายชื่อนี้ · ใส่ลิงก์รูปให้ NPC มีรูปเฉพาะตัว (เว้นว่าง = วงกลมสีอักษรแรก)
สร้างโพสต์อัตโนมัติ
โหมด "ทุกๆ จำนวนข้อความ" = ครบตามจำนวนแล้วสร้างเลย · "ให้ AI ตัดสินใจเอง" = ครบแล้ว AI เช็คก่อนว่าควรโพสต์ไหม · นับจากตอนเปิดแชท
ข่าวสาร
กดปุ่ม "สร้างข่าวใหม่" ในแท็บข่าวสารสร้างเองได้ · ข่าวปกติควรตั้งจำนวนข้อความสูงกว่าโพสต์ (ข่าวไม่ควรมาถี่) ข่าวถูกตัด/ยาวเกิน = เพิ่ม token · ตั้งค่าน้อยไป = ข่าวสั้น
แทรกฟีดเข้าประวัติแชท
เปิดแล้วตอน AI สร้างเนื้อหาในแอปหนึ่ง จะเห็นเนื้อหาแอปอื่นด้วย เช่น คุยในแชตแล้วพูดถึงโพสต์ในฟีด หรือผู้ชมไลฟ์พูดถึงข่าว (ใช้ token เพิ่ม) แทรกโพสต์/ข่าวล่าสุดเข้า context ของ RP หลัก → ตัวละครอ้างอิงโพสต์ได้ (เช่น "เห็นที่โพสต์แล้วนะ") · depth = แทรกลึกกี่ข้อความจากล่าสุด (ต่ำ = ใกล้ปัจจุบัน)
คอมเมนต์
ตอนสร้างโพสต์ใหม่ (ทั้ง AI สร้างและเราโพสต์เอง) จะมีคอมเมนต์ NPC ติดมาตามนี้ · โพสต์เราจะได้ยอดไลค์สุ่มด้วย (ใช้ generation เพิ่ม = เปลือง token ขึ้น)
